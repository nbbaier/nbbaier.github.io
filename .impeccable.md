## Design Context

### Users

Primary visitors are people evaluating Nico Baier professionally or academically: hiring teams and engineers, linguistics faculty and students, and collaborators. They typically arrive to confirm background, skim papers and dissertation metadata, download a resume or CV, and find contact links. The page should support fast scanning, credible first impression, and low-friction access to PDFs and external profiles.

### Brand Personality

**Voice inferred from implementation:** precise, approachable, and credibly academic without feeling stiff—warm off-white ground (`#fffffc`), restrained palette, monospace-flavored meta lines (venues, dates) that signal technical care. **Three-word handle (provisional—replace after you confirm):** _clear_, _scholarly_, _engineered_.

**References / anti-references:** Not stated in-repo; add explicit reference sites and “do not look like X” notes when you have them.

### Aesthetic Direction

- **Visual tone:** Editorial single-column feel inside a max-width container (~52rem); two-column section grid on desktop (label column + content), collapsing at **768px**. Light theme only in CSS today; primary text `#111` on warm paper.
- **Typography:** Default stack uses **Figtree** (body) and **Recursive** (headings and mono-axis metadata). Alternate **“Original”** preset (`body.typeset-original`) shifts toward a uniform **Geist Mono**–first stack (font not loaded in HTML; falls back to system UI fonts) with adjusted scale and muted color merged with primary.
- **Accent / chrome:** Hairline section dividers (`::after` rules), 2px header rule, fixed **design switcher** (pill, light border, subtle shadow)—functional, not decorative excess.
- **Motion:** Short transitions on the switcher only; no broad motion system in CSS.

### Design Principles

1. **Structure before decoration** — Section rhythm comes from the grid and horizontal rules, not cards or heavy backgrounds.
2. **Typography is the brand** — Recursive’s axes and Figtree’s readability are the main personality; the alternate typeset is a deliberate second voice, not an afterthought.
3. **Accessible disclosure** — Expandable content (e.g. dissertation abstract) uses `<button>`, `aria-expanded`, `aria-controls`, and `hidden` on the target; avoid native `<details>`/`<summary>` here due to prior layout regressions.
4. **Caret and control polish** — Paired inline SVG carets in one grid cell; visibility swaps via `[aria-expanded]` in CSS; no extra JS for icons; avoid `all: unset` on compact icon buttons.
5. **Ship as static files** — No build pipeline; HTML and CSS should remain GitHub Pages–friendly and easy to diff.
